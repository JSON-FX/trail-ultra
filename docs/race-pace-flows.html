trail·ultra

A multi-organization platform for trail & ultra events in the mountains of Mindanao. One runner account, many organizations — each with its own events, money, and forms, sealed off from every other.

iOS-first · Expo Supabase · RLS PayMongo getdesign · apple
MVP scope · what ships first

The MVP is two surfaces

The runner's mobile app and the admin web ship first — together they can run a real event end to end. The public e-commerce storefront is parked for later.

Flow 01

The runner journey — org-first

From install to start line, a runner is always inside one organization. The one thing the app never does is trust a payment redirect — it waits for PayMongo's webhook.

Flow 04

Who can do what

One runner role is global; staff roles are bound to a single organization; only the platform operator sees across all of them.

RoleScopeWhat they touch
user
the runner
 Global Browse, register, pay, carry a ticket — reads/writes only their own data.
marshal Org · often 1 event Scan & check in; look up an entry. Nothing editable, nothing financial.
editor Org Create & manage events, categories, add-ons, and custom fields; view registrations.
admin Org Everything editor does + registrations, refunds, check-in, and the org's settlement statements.
super_admin
platform operator
 Platform · all orgs Provision organizations, set commission, run payouts, manage roles, own global settings.
